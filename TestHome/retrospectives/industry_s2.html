Action Points

Information Sharing

  • The team decided more information may be useful and suggested that the scrum master should give a rundown of how many scripts are Not Started/ In Progress by priority.

Scripts

  • Tormod will ask CCP if they can send us extra information when giving us new scripts such as a list of defects already affecting the script and more context of the intended functionality.
  • Tormod will be disseminating information from the EVE dev blogs so the team has a reference of the intended functionality for the new scripts.

Script Changes

  • Only outdated terminology, values and spelling errors that are confirmed by a CCP source can be immediately implement.
  • Only one script change should be posted and then confirmed in the script change channel at a time.
  • The change request document can be temporarily shared when there are a lot of script changes. Testers will make sure to claim cells for their changes saving before and after to minimize accidental overwrites.

Scrum

  • The "Concerns within the team" section will be renamed to "Internal Issue" as this is a broader term.
