External Issues

DDOS Attacks

  • Attacks caused interruptions to the sprint.
    If an attack happens in the future, we should use the bypass to get back online faster.

CCP

  • The testing server was switched from Chaos to Singularity in the middle of the sprint.
    In future we could request to finish a few test steps before switching server so no setups have to be repeated.
  • Downtime requests were not always carried out on time or happened without prior notice.
    This is not something we can do anything about, CCP QA/Programmers have their own workload to complete which can result in delayed or unscheduled downtimes.
  • CCP contacted a tester personally to carry out a task.
    We should carry out the requested tasks, but inform the lead tester first.
    If this continues to happen regularly, the lead may need to have discussion with CCP to set boundaries.
  • Chaos clocks were out of sync which caused issues with some of the scripts.
    Daniel will be our contact if this happens again as he has dealt with this issue on previous occasions.

PTW

  • Testers have been pulled from the sprint to help with other projects
    As long as this does not affect the completion of the sprint then this is absolutely fine.
